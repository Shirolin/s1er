隐私政策

最后更新：2026-07-15

S1er（以下简称「本应用」）是第三方开发的 Stage1st（S1）论坛非官方客户端。本隐私政策说明本应用如何收集、使用和保护用户信息。

1. 信息收集与使用

1.1 不收集的信息

  • 密码：登录时密码仅在用户设备与 Stage1st 服务器之间直接传输，本应用不会记录或存储明文密码。
  • 个人身份信息：本应用不会收集姓名、电子邮件、手机号码等个人身份信息。
  • 设备信息：本应用不会收集设备型号、操作系统版本、唯一设备标识符等信息。
  • 位置信息：本应用不会访问或收集用户位置信息。
  • 使用数据：本应用不会向开发者服务器上报使用统计或分析数据。

1.2 本地存储的信息

以下信息仅存储在用户设备本地，不会上传至开发者或第三方服务器：

数据类型 存储方式 用途
登录 Cookie AES-256-GCM 加密文件 维持 Stage1st 会话状态
阅读历史与进度 Drift（本地 SQLite） 记录已读帖子和阅读位置
回复草稿 Drift（本地 SQLite） 保存未发送的回复内容
投票记录 Drift（本地 SQLite） 标记已参与的投票
黑名单 Drift（本地 SQLite） 屏蔽指定用户的内容
应用设置 Drift（本地 SQLite） 主题色、字号等偏好设置
图片缓存 本地磁盘缓存 加快图片加载速度

1.3 网络请求

本应用发起的网络请求仅限于：

  • Stage1st 论坛 API（stage1st.com）：获取论坛数据、发帖、登录等核心功能
  • Stage1st 头像 CDN（avatar.stage1st.com）：加载用户头像
  • 外链图床（p.sda1.dev）：上传帖子插图（用户主动操作时）
  • Sentry（可选）：仅在用户通过 --dart-define 注入 DSN 时启用，用于崩溃报告

2. 第三方服务

2.1 Sentry（可选）

本应用集成了 Sentry 用于崩溃监控。默认不启用，仅当用户编译时通过 --dart-define=SENTRY_DSN=... 显式注入 DSN 后，Sentry 才会初始化。启用后，Sentry 可能收集以下信息：

  • 崩溃栈追踪（代码位置）
  • 设备操作系统版本与型号
  • 应用版本号

Sentry 的数据处理受 Sentry 隐私政策 约束。

3. 数据共享与披露

本应用不会：

  • 向第三方出售用户数据
  • 向第三方分享用户个人信息
  • 将本地存储的数据上传至开发者服务器

4. 数据导出与删除

用户可以通过以下方式管理自己的数据：

  • 备份导出：在应用设置中使用「数据备份」功能导出阅读历史、设置等为 ZIP 文件
  • 清除数据：卸载应用将删除所有本地存储的数据
  • Cookie 清除：在应用中退出登录将清除已存储的登录 Cookie

5. 儿童隐私

本应用面向 Stage1st 论坛用户，不面向 13 岁以下儿童。本应用不会故意收集儿童的个人信息。

6. 变更

本隐私政策可能不时更新。变更后的版本将通过应用更新或 GitHub 仓库公告通知用户。

7. 联系方式

如有关于本隐私政策的疑问，请通过以下方式联系：

  • GitHub Issues： https://github.com/Shirolin/s1er/issues
